# GENEROUS rating rubric (v2)
This is a re-rate to reduce excessive blanks. Read ALL fields (Type, One-Liner, Target Segment, Key Capabilities, Weaknesses, Vertical). Infer reasonably — if a competitor operates in an area, credit the related features rather than leaving them blank.

- "Strong"  = the competitor clearly delivers this capability as a core, productized offering.
- "Partial" = the competitor plausibly touches / supports / is adjacent to this capability, OR it operates in a workflow area where this artifact lives. Be generous here — when in doubt between Partial and blank, choose Partial if there is ANY reasonable connection.
- (omit)     = genuinely no connection at all.

## Mapping hints for adjacent categories (so they aren't all blank)
- Patient recruitment / patient identification / matching / enrollment tools → "Recruitment Materials (advertisements, flyers)" and "Subject Screening & Enrollment Log" (Partial, or Strong if recruitment IS the core product).
- Site selection / feasibility / site networks → "Site Feasibility & Selection" (Strong if core) + "Site Feasibility Questionnaires" (Partial).
- EDC / data capture / data management platforms → "CRF Generation" + "Data Management Plan (DMP)" (Strong/Partial) and often "Subject Screening & Enrollment Log" (Partial).
- Trial databases / intelligence / RWE / analytics → "Trial Search & Intelligence", "Trial-Specific Information Retrieval", "Competitive Analysis", "Comparative Analysis & Insights".
- Regulatory writing / medical writing / dossiers (CTD, CER, PSUR, eCTD, narratives) → "Regulatory Planning", "Submission Cover Letters (Regulatory & Ethics)", "Clinical Study Report (CSR)", "IND for FDA" (Partial+).
- Pharmacovigilance / safety / literature monitoring → "Trial Search & Intelligence" / "Trial-Specific Information Retrieval" (Partial).
- Protocol design / study design / simulation / optimization → "AI-Driven Protocol Optimization", "Protocol Generation", "Comparative Analysis & Insights".
- Document/archiving/eTMF/compliance tools → "Archiving & Record Retention", "Notes to File", "Monitoring Plan".

Even thin entries usually deserve at least 1–2 Partial ratings based on their Type/Vertical. Only leave a competitor completely empty if there is truly nothing in any field to go on.
